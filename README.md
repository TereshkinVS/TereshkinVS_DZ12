## Описание

В рамках работы выполнен анализ сетевого дампа `dhcp (1).pcapng`.  
Задача: извлечь ключевые артефакты (IP-адреса клиентов и сервера DHCP, типы DHCP-сообщений), построить минимальную визуализацию и сохранить результаты анализа.
## Используемые инструменты

- Python 3
- Scapy
- pandas
- matplotlib
- seaborn
- Google Colab (для выполнения ноутбука)

## Как запустить анализ

1. Откройте ноутбук `dhcp_analysis.ipynb` в Google Colab или локальном Jupyter.
2. Установите зависимости (ячейка с `pip install scapy pandas matplotlib seaborn`).
3. Загрузите файл дампа `dhcp (1).pcapng` (в Colab через `files.upload()`).
4. Выполните ячейки с анализом:
   - извлечение DHCP-пакетов,
   - вывод сводной статистики,
   - построение графиков,
   - сохранение `dhcp_analysis.csv`.

## Краткий отчёт

Источник данных:  
- Файл сетевого дампа `dhcp (1).pcapng`, содержащий DHCP-трафик одной сессии.

Метод анализа:  
- Чтение дампа с помощью Scapy, фильтрация пакетов с слоями IP/UDP/BOOTP/DHCP.  
- Извлечение полей: `src_ip`, `dst_ip`, `yiaddr` (IP клиента), `siaddr` (IP сервера), `xid`, `message-type`.  
- Построение временного распределения DHCP-сообщений и частот IP-адресов клиентов.

Найденные артефакты (пример — замените на свои реальные значения):  
- DHCP-сервер: `192.168.0.1` (4 сообщения).  
- Клиентский IP: `192.168.0.10` (получен 1 раз).  
- Типы сообщений: DISCOVER, OFFER, REQUEST, ACK (по одному каждому).

Предварительный вывод:  
- Зафиксирована полноценная DHCP-сессия, поведение сервера и клиента соответствует ожидаемому.  
- Аномалии в работе DHCP-сервера и распределении адресов не выявлены.

Рекомендации:  
- Продолжить мониторинг DHCP-логов и сетевых дампов при расследовании инцидентов.  
- При появлении повторяющихся запросов DISCOVER/REQUEST без ACK проверить конфигурацию сервера и пул адресов.
